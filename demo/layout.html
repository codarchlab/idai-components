Major layout elements

Navbar

Example

See top of the page.

Usage

        <idai-navbar
            project-id="projectId"
            login-function="loginFunction()"
            logout-function="logoutFunction()"
            user-object="user"
            hide-register-button="true"
            hide-contact-button="true">
        </idai-navbar>

Pass a unique project identifier in project-id. This identifier is used to construct the filename of a project logo in the img directory. The pattern is logo_project-id.png.

Via login-function and logout-function it is possible to bind functions from the surrounding scope to the sign in and logout (hidden in dropdown) buttons buttons. Pass the user object from the surrounding context

into user-object. When not empty, the sign in and sign up buttons get replaced by a user dropdown menu which also contains the logout button.

The hide-register-button and hide-contact-button properties can be used to hide the respective elements. If set to false or the properties are not set at all, the elements are shown.

The navbar requires the application to provide some html which will get rendered seamlessly into the directive's html:

  • partials/navbar-menu.html - will be inserted as html into the dropdown menu
  • partials/navbar-project.html - will be inserted as brand text next to your projects logo.

The navbar auto-generates its link titles with help of the content localization system described here. The configuration of the shown links is required to be found at info/content.json. The localization system will search for a subtree with the id "navbar".

Footer

Example

See bottom of the page.

Usage

<idai-footer></idai-footer>

Use mailto to pass an eMail address that should appear as a contact for bugreports in the footer.

The footer auto-generates its link titles with help of the content localization system described here. The configuration of the shown links is required to be found at info/content.json. The localization system will search for a subtree with the id "footer".

The email address for bug reports can be configured via componentsSettings.mailTo.

Content you fill in between the opening and the ending idai-footer tag is going to be displayed in the third row of the footer.
Let the tag content free to have the standard-license information "Licensed under Creative Commons" displayed in the footer.
The footer you can see below is generated from the following markup: 
        <idai-footer
            institutions="{'dai': 'http://www.dainst.de/',
                'unikoeln': 'http://archaeologie.uni-koeln.de',
                'codarchlab': 'http://archaeologie.uni-koeln.de/node/23'}">
                GitHub-Repository: https://github.com/dainst/idai-components
        </idai-footer>

Images

Required artifacts (which can be found in the local img folder of this repo):

  • img/kleinergreif.png - will appear in the navbar.
  • img/logo_unikoeln.png - will appear in the footer
  • img/logo_codarchlab.png - will appear in the footer
  • img/logo_dai.png - will appear in the footer

Header

idai-components provides the directive <idai-header> to extend your project pages with a responsive and interactive header background.

Example

See top of the page, right beneath the navbar.

Usage

<idai-header
    image="/img/exampleteaser.jpg"
    description="Palmyra, Baal-Heiligtum, Bankettsaal"
    link="http://arachne.dainst.org">
</idai-header>

<div class="teaserpage teaserbgr container">
    <main>
        <h1>Project Title</h1>
        Page content. Here, div-container with bootstrap's classes can be used to structure your content.
    </main>
</div>
The <idai-header> takes three arguments:
  • image - image to be shown as header background. The header image you can see on this page has a dimension of 1680 x 676 pixel and a file size of 120 KB.
  • description (optional) - a short description of the image.
  • link (optional) - you can provide a url to connect the header with a web ressource.
Wrap your page content in a div-Container with the classes teaserpage and container or use the predefined class con10t-page to have your page rendered correctly.

Extended example

The following example page uses con10t classes. Be sure to add the class teaserpage to your con10t-page: 
<idai-header
    image="/img/exampleteaser.jpg"
    description="Palmyra, Baal-Heiligtum, Bankettsaal"
    link="http://arachne.dainst.org">
</idai-header>

<div class="con10t-page teaserpage container">
    <div class="con10t-row">
        <div class="con10t-title">
            <img src="path/to/your/project-logo" class="img-responsive">
        </div>
        <div class="con10t-sidebar">
            Sidebar content
        </div>
        <div class="con10t-content">
            Main content
        </div>
    </div>
</div>
